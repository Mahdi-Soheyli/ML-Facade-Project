Common algorithm names and how they relate to this app

  • k-nearest neighbors (k-NN) — Predicts a numeric target by averaging (or weighting) the k training examples closest in feature space. This is what the dashboard implements (several k values for comparison). Good fit when labels come from a deterministic “oracle” and you want a simple, interpretable interpolating model.
  • Regression (linear, ridge, polynomial, etc.) — Learns an explicit function from features to thickness or resistance. Appropriate in principle for the same prediction task; not the current implementation. Often needs more tuning and care with nonlinear geometry.
  • Support Vector Machine (SVM) — In regression form (SVR), it can predict continuous values from features; the classic SVM story is often classification. Possible alternative learner; not used here. k-NN was chosen for transparency in teaching.
  • K-means — Unsupervised: partitions points into k clusters without using oracle labels the same way. The thickness “clusters” in section 4 come from the oracle Table 4 mapping, not from K-means. You could use K-means on feature space for a different grouping exercise; that would be a separate pedagogical choice.

Summary: for predicting resistance/thickness from panel features with training labels from this codebase, k-NN regression is the algorithm family implemented; the others are useful context or alternatives.
