# E1300 Teaching API (ML Facade)

FastAPI service that combines **wind-derived design pressure**, an **ASTM E1300-24** load-resistance **oracle**, and a **k-nearest neighbors** ML layer on top of oracle-generated training data. It ships with a browser **dashboard** (static HTML) for teaching and demos.

**Disclaimer:** This project is for education and engineering exploration. Always verify results against the published standard and applicable building codes for real projects.

## Features

- **Wind:** Terrain and elevation inputs; design pressure in kPa for panels.
- **Oracle:** Deterministic E1300-style logic (tables, GTF/LSF/NFL, minimum nominal thickness).
- **ML:** KNN predictions using bundled NumPy arrays (training pipeline in-repo; inference runs without scikit-learn in the Docker image).
- **Sessions:** Upload panel geometry, set wind via API or UI, run analysis, optional clustering for visualization.
- **Grasshopper:** Example GhPython scripts post JSON to the HTTP API (see [docs/GHPYTHON.md](docs/GHPYTHON.md)).

## Repository layout

| Path | Purpose |
|------|---------|
| `app/` | FastAPI app, wind/oracle/ML/clustering, static dashboard assets |
| `e1300/` | E1300 numerical helpers (NFL surrogate, tables, LR) |
| `building_code/e1300_data/` | JSON table data and chart metadata |
| `models/` | Trained ML bundle (`ml_bundle.npz`, `ml_meta.json`, `ml_registry.json`) |
| `scripts/` | Training (`train_ml.py`), Grasshopper client examples, utilities |
| `tests/` | `pytest` suite (API, geometry, Annex A3 checks) |
| `tools/` | One-off helpers (e.g. chart image extraction) |
| `docs/` | Dataset documentation and Grasshopper HTTP examples |
| `maintainer/` | Optional hosting notes (e.g. Railway checklist); safe to omit from a public clone if you prefer |

Large CSV datasets and Rhino working files are described in [docs/DATASET.md](docs/DATASET.md). Sample modeling assets belong under `modeling_files/` (ignored by git by default).

## Requirements

- **Python 3.11+** recommended (matches the Docker image).
- Dependency groups:
  - [requirements.txt](requirements.txt) — NumPy, scikit-learn, PyMuPDF, pytest, etc.
  - [requirements-api.txt](requirements-api.txt) — FastAPI stack for the web service.
  - [requirements-train.txt](requirements-train.txt) — Minimal deps for `scripts/train_ml.py`.

Install for local development:

```bash
python -m pip install -r requirements.txt -r requirements-api.txt
```

## Run locally

From the repository root:

```bash
uvicorn app.main:app --reload --host 127.0.0.1 --port 8080
```

Then open [http://127.0.0.1:8080/](http://127.0.0.1:8080/) for the dashboard, or [http://127.0.0.1:8080/docs](http://127.0.0.1:8080/docs) for OpenAPI.

## Docker

```bash
docker build -t e1300-api .
docker run --rm -p 8080:8080 e1300-api
```

The image listens on `0.0.0.0` and uses the `PORT` environment variable when set (for example on cloud hosts).

## Dataset generation and ML

- **Research CSV:** Generated by `generate_data.py` using the oracle. Details: [docs/DATASET.md](docs/DATASET.md).
- **Retrain ML bundle:** After installing `requirements-train.txt`, run `python scripts/train_ml.py` and commit updated files under `models/` if you change training data or labels.

## Tests

```bash
python -m pip install -r requirements.txt
pytest
```
